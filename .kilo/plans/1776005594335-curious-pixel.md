# Plan: Tối ưu Base ant-design-pro theo mô hình bmad/seo-side

## Mục tiêu
Tối ưu cấu trúc ant-design-pro bằng cách đồng bộ hóa với mô hình @seo-side/, giữ nguyên UI/UX tối ưu của Ant Design, sử dụng các skill BMad để đạt kết quả tốt nhất.

---

## Phase 1: Discovery & Analysis (Khám phá & Phân tích)

### 1.1 Document Current State
**Skill:** `bmad-document-project`
**Mục đích:** Tạo tài liệu chi tiết về cấu trúc hiện tại của ant-design-pro

**Output:**
- Inventory tất cả files/folders
- Phân loại: business logic modules vs UI-only modules
- Dependencies analysis
- Technical debt assessment

### 1.2 Analyze Gap với seo-side
**Skill:** `bmad-agent-analyst` (Mary)
**Mục đích:** So sánh chi tiết giữa hai cấu trúc

**Output:**
- Gap analysis report
- Which layers/features are missing
- Which can be simplified/removed

## Phase 2: Ideation & Planning (Ý tưởng & Lập kế hoạch)

**Topics:**
1. Cách đơn giản hóa structure mà không mất UI/UX
2. Module migration strategy (business vs UI-only)
3. Files/folders có thể xóa an toàn
4. Layer architecture adoption approach

### 2.2 Create Architecture Design
**Skill:** `bmad-create-architecture` (Winston)
**Mục đích:** Thiết kế target architecture

**Output:**
- Target folder structure
- Core infrastructure design (base-api, base-service, base-hooks)
- Module layer strategy
- Migration sequence

### 2.3 Create Epic & Stories
**Skill:** `bmad-create-epics-and-stories`
**Mục đích:** Break down thành các stories có thể execute

**Output:**
- Epic: Structure Optimization
- Stories: Theo từng phase/module
- Acceptance criteria rõ ràng

│   ├── constant/

### UI-only Modules (Giữ nguyên structure, không cần migrate layers)
- dashboard, form, list, table, exception, result
- **Lý do:** Không có business logic, chỉ là UI components đơn giản
- **Xử lý:** Giữ nguyên structure hiện tại, có thể đơn giản hóa sau

### Clarification Needed

**Question:** Các UI-only modules (dashboard, form, list, table, exception, result) hiện tại có đang có service/hooks layers không, hay chúng là đơn giản chỉ là pages với API calls trực tiếp?

- **Nếu có service/hooks layers:** Có cần migrate chúng sang đúng layer pattern (UI → Hooks → Service) không?
- **Nếu chỉ là pages đơn giản:** Có thể giữ nguyên không cần thay đổi (vì không có business logic phức tạp)

### Files/Folders có thể XÓA:
- `src/services/` - Demo code từ ant-design-pro template
- `mock/` - Nếu không dùng mock API
- `tests/` - Di chuyển vào từng module nếu cần
- `docs/` - Di chuyển ra ngoài src/
- `cloudflare-worker/` - Nếu không dùng Cloudflare Workers
- `design-artifacts/` - Di chuyển ra ngoài src/
