# Kế hoạch Refactor Modular Architecture

## Mục tiêu
Chuyển đổi cấu trúc project từ Layer Architecture sang Modular Architecture:
- Mỗi module gồm 5 folder: `pages`, `components`, `hooks`, `services`, `types`
- Không cần index.ts (export các file trong module)
- Sử dụng path alias trong tsconfig.json để import

## Hiện trạng
- Project đã có `src/modules/` với 10 modules (account, auth, dashboard, exception, form, list, profile, result, table, user)
- Các module đã có sẵn 4 folder: pages, components, hooks, services (trừ dashboard chỉ có hooks)
- Pages hiện tại nằm trong `src/pages/` với layer architecture
- Route được định nghĩa trong `config/routes.ts`

## Kế hoạch thực hiện

### Bước 1: Cập nhật tsconfig.json
Thêm path alias cho modules:
```json
"paths": {
  "@/modules/*": ["./src/modules/*"]
}
```

### Bước 2: Tạo folder `types` cho các module hiện có
Tạo folder `types` trong 9 module (trừ dashboard vì chưa có gì):
- src/modules/account/types
- src/modules/auth/types
- src/modules/exception/types
- src/modules/form/types
- src/modules/list/types
- src/modules/profile/types
- src/modules/result/types
- src/modules/table/types
- src/modules/user/types

### Bước 4: Cập nhật routes.ts
Cập nhật component path trong config/routes.ts:
- `./dashboard/analysis` → `@/modules/dashboard/pages/analysis`
- `./form/basic-form` → `@/modules/form/pages/basic-form`
- ... (tương tự cho tất cả routes)

### Bước 5: Kiểm tra và chạy build
- Chạy `npm run build` hoặc `npm run dev` để verify
- Fix các lỗi import nếu có

### Bước 6: Xóa src/pages/*
Sau khi migrate hoàn tất và build thành công, xóa src/pages/

## Cấu trúc module sau refactor
```
src/modules/{module-name}/
├── pages/          # Chứa các page components
│   └── {page-name}/
│       └── index.tsx
├── components/     # Chứa các components riêng lẻ
├── hooks/          # Gọi đến service
├── services/       # Call API sử dụng request từ @umijs/max
└── types/          # TypeScript definitions
```

## Lưu ý
- Service files giữ nguyên `import { request } from '@umijs/max'`
- Giữ nguyên các file styles (_mock.ts, style.less, style.style.ts) cùng với component
- Không tạo index.ts export trong mỗi module
- Sử dụng path alias `@/modules/...` trong routes.ts
