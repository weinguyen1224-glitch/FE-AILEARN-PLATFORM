# So sánh Base Hooks: React Query vs UMI useRequest

## 1. Hiện trạng project

**Project đang dùng:** `@umijs/max` với `useRequest`

**File hiện tại:** `src/config/hooks/base.hooks.ts`

---

## 2. Bảng so sánh

| Tiêu chí | Format React Query | Format UMI useRequest |
|----------|-------------------|----------------------|
| **Library** | @tanstack/react-query | @umijs/max useRequest |
| **Typing** | `UseQueryResult<T>`, `UseMutationResult<T, Error, TVariables>` | `any` (không strongly typed) |
| **Query Key** | Có queryKey rõ ràng | Không có queryKey (UMI dùng service URL) |
| **Cache Invalidation** | Tự động `onSuccess` + `queryClient.invalidateQueries` | Thủ công, phải gọi `refresh()` |
| **Optimistic Update** | Hỗ trợ `setQueriesData` trong `onSuccess` | Không có sẵn |
| **Automatic Refetch** | Có `staleTime`, `refetchInterval` options | Có nhưng ít tùy chọn hơn |
| **Devtools** | React Query Devtools tích hợp | Không có |

---

## 3. Điểm yếu của format UMI hiện tại

```typescript
// ❌ Hiện tại: không có cache invalidation tự động
const useCreate = () => {
  return useRequest((data: TCreateDto) => service.create(data), {
    manual: true,
  });
};

// ❌ Query options là any - không có type safety
const useGetPage = (options?: QueryOptions, queryOptions?: any) => {
  return useRequest(() => service.getPage(options), {
    ...queryOptions,
  });
};
```

### Vấn đề cụ thể:
1. **Không có cache invalidation** - Sau khi create/update/delete, data không tự động refresh
2. **Typing yếu** - `queryOptions?: any` không có gợi ý type
3. **Không có optimistic update** - Không thể cập nhật UI ngay lập tức
4. **Hardcoded string "sanPham", "TaiLieu"** - Lẫn lộn trong predicate

## 4. Khuyến nghị cải tiến

### Option A: Giữ UMI nhưng cải tiến

```typescript
// Cải tiến: Thêm cache refresh tự động
const useCreate = () => {
  const { refresh } = useRequest(() => service.create(data), {
    manual: true,
  });
  
  return {
    ...useRequest((data: TCreateDto) => service.create(data), {
      manual: true,
    }),
    mutateAsync: async (data: TCreateDto) => {
      const result = await service.create(data);
      refresh(); // Invalidate cache
      return result;
    }
  };
};
```

**Ưu điểm:** Giữ nguyên UMI, thêm cache handling
**Nhược điểm:** Không tận dụng được full features của react-query

### Option B: Chuyển hoàn toàn sang React Query

**Ưu điểm:**
- Typing hoàn chỉnh
- Cache invalidation mạnh mẽ
- Optimistic updates
- Devtools
- Stale-while-revalidate mặc định

**Nhược điểm:**
- Phải thay đổi cơ chế từ đầu
- UMI request có sẵn nhiều tính năng tích hợp ( SSR, auth )

### Option C: Hybrid approach (Khuyến nghị)

Giữ `useRequest` cho những thứ UMI handle tốt (request thông thường), nhưng bổ sung:

```typescript
// Tạo wrapper cho useRequest để thêm cache management
const useCreate = (options?: { onSuccess?: () => void }) => {
  return useRequest(
    (data: TCreateDto) => service.create(data),
    {
      manual: true,
      onSuccess: () => {
        // Gọi refresh hoặc invalidate
        options?.onSuccess?.();
      },
    }
  );
};
```

## 5. Kết luận

| Nếu project cần | Nên dùng |
|----------------|----------|
| SSR, Auth tích hợp sẵn | UMI useRequest (cải tiến) |
| Cache phức tạp, optimistic UI | React Query |
| Nhanh, đơn giản, không cần nhiều cache | UMI useRequest |

**Đề xuất:** Nên cải thiện typing cho UMI version trước, sau đó nếu cần cache phức tạp thì chuyển sang React Query.

## 6. Các bước thực hiện Option A: Cải tiến UMI useRequest

### Step 1: Cải thiện Typing cho queryOptions
Thay `any` bằng `UseRequestOptions` từ UMI

```typescript
import { useRequest, UseRequestOptions } from '@umijs/max';

const useGetPage = (
  options?: QueryOptions, 
  queryOptions?: UseRequestOptions<PageableResponse<TEntity>>
) => {
  return useRequest(() => service.getPage(options), queryOptions);
};
```

### Step 2: Thêm Cache Invalidation tự động
Tạo helper để handle cache invalidation:

```typescript
const createMutateHook = (mutationFn: Function) => {
  return (options?: { onSuccess?: () => void }) => {
    const { refresh } = useRequest(() => ({}), { manual: true });
    
    return useRequest(mutationFn, {
      manual: true,
      onSuccess: () => {
        refresh();
        options?.onSuccess?.();
      }
    });
  };
};
```

### Step 3: Cố định Bug hardcoded string
Kiểm tra và sửa các chuỗi "sanPham", "TaiLieu" trong predicate functions

### Step 4: Thêm helper types
Tạo type cho mutation results để đảm bảo consistency
